LIMITS OF EXCEL

It is quite common to find organisations that use Microsoft Excel’s as their database for storing their information. Despite Excel being one of the most powerful tools of the Microsoft Office’s family, it has never been really designed for such a purpose. Yet, things can still be accomplish by using Excel as your database but things can be become unmanageable as your volume of data grows along with the complexity of your tasks. The following are some disadvantages and limitation of using Excel as a database:

  • It can require skilled users with specific knowledge and understanding of how spreadsheet works. Also to achieve more complex tasks the user needs to be familiar with VBA, a programming language that runs within all products of the Office Family.

  • It cannot ensure data's integrity and it prones to redundancy among cells and spreadsheets. If some data should be changed the user must ensure that those changes are reflected wherever is necessary within the file.

  • It is human error-prone, which means the user can easily insert the wrong information and realise it only later.

  • Compatibility issues might arise when the file is moved from one computer to another.

  • Measures to correct some of the aforesaid limitations exists; However, it can become extremely complicated and time-consuming to implement these correcting methods across spreadsheets and excel's files.

  • The aforesaid limitations will exacerbate if the file should be handled by multiple users.

BENEFITS OF A DATABASE

Databases are programs designed to store, manage and organise great volume of data. This technology started in 1960 and it has replace the use of files as a way of storing and retrieving information. Today they are an essential component of even the most simple application. However, a user does not deal with a database directly but always through some kind of user interface such as a Desktop, Mobile or Web Application.

Here are some benefits of using a Database:

  • reduces the amount of time spent for managing data.

  • data can be analysed in a variety of ways.

  • improves the quality and consistency of information

  • different bits of information can be put together producing more valuable pieces of information

  • the final user will have to deal only with a user interface and therefore it drastically reduces human-errors.

  • data can be access and shared with multiple people within your organisation or even worldwide through a website.

DATABASES I WORK WITH

Microsoft Access

Microsoft Access in a component of the Microsoft Office Family. It shares many similar features with Excel but overcome some of its limitations as this product is a real database. As a member of the Office Family, it can easily interact with its siblings such as Excel, Word, Outlook, and PowerPoint. Even though Microsoft Access requires a developer to be built, the final user will deal with the data only through an user interface that is built in Access.

Pros:

  • Good for small companies and individuals.

  • Fast to design and develop.

  • It comes with the Office Pack.

  • Reports are easy to design and export in both PDF and Excel.

Cons:

  • Not well designed for multi-users interactions.

  • Compatibility issues might arise when the file is moved to another computer.

  • Might not be suitable for more complex applications.

  • Can store only up to 2 Gigabytes of data.

SQLite

SQLite is one of the most common used databases in the world. Among to its features it is cross-platform and the does not require additional software to run on the computer. It does not come with an user interface but it can be easily accessed by any piece of software. It can store up to 140 terabytes of data and despite being small in sizes it is fast and highly reliable.

Even though SQLite supports multi-users interactions, it might not necessarily be the best choice in some circumstances. However, it works great as the database engine for most low to medium traffic websites. It is also a common choice for Desktop and Mobile applications.

MySQL

MySQL is the most common and largely used database on the internet. It is highly performant, reliable, and easy to use. It has no storage limit and each of its data containers (Tables) can store up to a maximum size of 256 terabytes per table. It is the number one choice for applications that deal with multiple user interactions.
